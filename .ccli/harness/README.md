# Harness 说明

ccli 的 Harness 是模型之外的外部支架，负责把智能体运行变成稳定流程：固定规则、限制工具、自动验证、受控返工、进度落盘和审计记录。

## 默认约束

- 模型可以参与规划、开发和审查，但用户界面只展示中文产品语义。
- 工具按阶段暴露，避免让模型在无关工具之间游移。
- 自动验证结果优先回流给开发代理，不直接推给普通用户判断。
- 每次任务都写入 `.ccli/progress.json`，长期经验写入项目记忆。

## 使用顺序

先让一次人工触发的任务稳定完成，再补权限、审查、技能、钩子和记忆；这些都稳定后，才把循环自动化放上去。

普通用户只需要说“补齐驾驭系统”“查看驾驭路线图”或“记住这次经验”，系统会把底层支架补齐并用中文解释当前缺口。
