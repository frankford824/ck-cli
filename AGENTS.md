# ccli 项目指南

这个仓库开发的是面向普通用户的全中文开发 CLI。所有改动都应优先降低学习成本，同时保留可审计、可验证、可回滚的工程流程。

## 用户体验

- 普通模式只展示中文自然语言。
- 不向普通用户展示代码、命令、路径、diff、堆栈或原始模型文本。
- 用户需要看到任务、进度、风险、选择和结果，而不是技术过程。
- 专家细节只在专家模式或本地审计日志中出现。

## 工程规则

- 先使用现有包和模式，再增加新抽象。
- 每个高影响动作都必须经过策略层和中文确认。
- 自动验证失败时，把失败反馈交回开发代理做一次受控修复。
- 每次任务都要更新本地进度，方便上下文丢失后接管。

## 交付规则

- 默认使用 git 分支、独立审查和 Draft PR 流程。
- 推送、合并、发布和部署不能在无确认的情况下静默执行。
- PR 文案面向团队审查，普通 CLI 输出仍保持中文产品语义。
